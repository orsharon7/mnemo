# Changelog

All notable changes to ClipMate. Format loosely follows [Keep a Changelog](https://keepachangelog.com/).

## [0.4.0] — 2026-05-11

### Added
- **On-device semantic search** via Apple's `NLEmbedding` (English sentence model,
  512-dim, L2-normalized). No network calls, no API keys.
- `Sources/Embedder.swift` — singleton wrapper with input trimming, 2000-char cap,
  and normalize/cosine helpers.
- `ClipEntry.vector: [Float]?` field — backward-compatible Codable
  (`decodeIfPresent`); old `history.json` upgrades cleanly.
- Background backfill at launch for legacy entries (batches of 50, persisted between).
- Hybrid search in `HistoryStore.search()`: substring → semantic top-K (threshold
  0.30, K=10) → fuzzy subsequence. Pinned entries always float to top.
- `Settings.semanticSearchEnabled` (default ON) + Preferences "Search" section.

## [0.3.1] — 2026-05-11

### Fixed
- Arrow keys, Esc, Return and the new ⌘P / Space / ⌘⌫ shortcuts were silently
  dead inside the search panel. Root cause: NSTextField's embedded field editor
  (NSText) consumes `keyDown` before reaching subclass overrides. Replaced the
  `ClipMateSearchField` subclass with a window-scoped
  `NSEvent.addLocalMonitorForEvents` installed by the SearchField Coordinator,
  gated on `field.window?.isKeyWindow`.

## [0.3.0] — 2026-05-11

### Added
- Pin/favorite entries (⌘P). Pinned entries sort first and survive retention
  trimming and Clear All.
- Type detection at capture (URL, email, JSON, code, multiline, text), surfaced
  as row badges in the panel.
- Quick Look preview on Space (when the search field is empty).
- First-launch onboarding (non-modal floating SwiftUI window — `NSAlert.runModal`
  froze the pasteboard watcher; see Decision #024).
- Secret blocker: known token prefixes (`sk-`, `ghp_`, `AKIA`, `eyJ`, `ya29.`,
  `xoxb-`, `glpat-`, …) plus high-entropy single-line strings (Shannon ≥ 4.2).
  Honors pasteboard hints from 1Password (`org.nspasteboard.AutoGeneratedType`,
  `org.nspasteboard.ConcealedType`, `org.nspasteboard.TransientType`).
- Cmd-Delete (with empty search) deletes the selected entry.

## [0.2.0] — 2026-05-11

### Added
- Persistent history at `~/Library/Application Support/ClipMate/history.json`
  (mode 0600, dir 0700) with retention by count and age.
- Configurable global hotkey, configurable auto-paste-on-Enter, launch-at-login
  via `SMAppService`.
- Preferences window (SwiftUI Form).
- Clear All with confirmation; per-entry copy / paste / delete actions.
- Source app capture (bundle id + display name) shown as subtle row metadata.

## [0.1.0] — 2026-05-11

### Added
- Initial spike. Menu-bar-only app (no Dock icon), `LSUIElement = true`.
- Pasteboard watcher (200 ms `changeCount` poll, sensitive-type filter).
- Global hotkey via Carbon (`RegisterEventHotKey`), default ⌥⌘V.
- Floating search panel (SwiftUI in NSPanel) with substring + subsequence fuzzy
  search, arrow-key navigation, Enter to copy, Esc to dismiss.
- Built without Xcode: `swiftc` + Makefile, ad-hoc codesigned.
